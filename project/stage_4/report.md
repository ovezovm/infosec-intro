## 1. Цель работы
Освоить работу с инструментом **Nikto** для выявления уязвимостей веб‑приложений и веб‑серверов, провести сканирование DVWA и проанализировать результаты.

---

## 2. Исходные условия
- **Целевая система:** DVWA (Damn Vulnerable Web Application)  
- **Уровень безопасности:** Low  
- **Метод авторизации:** HTTP POST Form  
- **Инструмент:** Nikto v2.5.0  

---

## 3. Выполнение работы
1. Запустить DVWA и убедиться в доступности веб‑приложения.  
2. Составить и выполнить команду Nikto:  
   ```bash
   nikto -h http://127.0.0.1
   ```
3. Дождаться завершения сканирования и зафиксировать найденные уязвимости.

**Figure 2:** найденные уязвимости *(место для скрина)*  

**Результаты сканирования:**  
- Выполнено 8074 запроса, найдено 26 потенциальных уязвимостей.  
- Обнаружены PHP‑бекдоры:  
  - `/wp-includes/js/tinymce/themes/modern/Meuhy.php?filesrc=/etc/hosts`  
  - `/assets/mobile/css/meta.php?filesrc=`  
- Найдены CGI‑скрипты с возможностью удалённого выполнения команд:  
  - `/cgi-bin/router`  
  - `/shell2.cgi`  
- Обнаружена версия Apache 2.4.65, отсутствующая в базе Nikto.

---

## 4. Вывод
Nikto выявил критические уязвимости, позволяющие удалённый доступ и выполнение команд.  
Для повышения безопасности необходимо:
- удалить вредоносные файлы;
- обновить ПО;
- ограничить доступ к служебным ресурсам.
